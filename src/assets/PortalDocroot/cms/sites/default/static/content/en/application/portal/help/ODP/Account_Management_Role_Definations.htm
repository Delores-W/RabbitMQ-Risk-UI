Account Management - Role Definations

Purpose:

This screen facilitates editing and creating of roles for all applications. It assists in defining Portal functionalities (i.e. the navigation links) that can be accessed for each role.

The accessibility defined on this screen for a particular role, will be applicable to all the users who have been assigned the same role.

Actions:

  • Select the application to define roles to restrict access.
  • Select the Role for the application.
    • Existing Role for modifications
    • Add New Role- to create a new role

Existing Role

  • Data is pre-populated and the permissions are fetched based on the Role Name selected.
  • Application and User role fields are editable from the dropdown menus.
  • Select or deselect check boxes to define the access.
  • Click on Save Changes to save all the changes made.
  • Click on Cancel Changes to discard all actions.
  • Click on Delete Role to remove the selected role.

Add New role

  • Make the various selection by choosing/entering the relevant options
    • New Role Name: Enter the role name to be created (maximum word limit 10 characters)
    • Role Type: Select the desired option from the menu.
    • Can be Reassigned: Choose the valid radio button.
    • Description: Enter the description for the role being created

Tips:

  • The Portal functionality would be generated based on the Role Type Selected by the user.
  • Click on Save Changes to update the lists.
  • Click on Cancel Changes to discard all actions.
