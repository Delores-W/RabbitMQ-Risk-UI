Portal Configuration - Schedule Down Time

Purpose:

This screen displays the list of Downtimes; either scheduled or completed and allows User to schedule, edit or delete down time.

Once the Downtime has started only the End date / Time and the Reason can be changed.

Actions:

  • Click on New to schedule a new down time. The Down Time Tab will be provided.
    • Enter the duration of the down time to be scheduled (date and time).
    • Enter the reason for the down time that will then be displayed in the list above.
    • Select the application from the drop down menu that will be unavailable during the down time.
    • Click on Schedule ; the Down time then appear in the master list with the status scheduled.
  • Click on Update to edit a Scheduled downtime by selecting the relevant radio button.
    • The Down Time Tab would be prepopulated with the data, user may update the fields.
    • Click on Save Changes to update the modifications.
  • Click on Completed to update the status of a completed downtime (please select the relevant radio button).

  • Select a radio button and click on Delete to remove the downtime from the list.

Tips:

  • In case of Scheduling new down time, if the date/time entered is incorrect/expired, an error message is displayed

    Invalid date/time range

  • The records can only be deleted if the Status is completed , else an error message is displayed

    Record can be deleted only after it is marked as completed

  • Completed downtime cannot be Updated.
  • To view the next page, click on next button seen on the bottom right corner of the display. To view a specific page, click on page number.
